### 4. `renderDataSpec()` に type `'C'` と `'I'` の寸法グリッドを追加する

現在は `type === 'H'`・`'L'`・`'FL'` のみ分岐している。以下を追加する。

**溝形鋼 (type === 'C') の寸法グリッド：**

表示する寸法ラベル：
- H（ウェブ高さ）、B（フランジ幅）、t1（ウェブ厚）、t2（フランジ厚）、r1、r2

**I形鋼 (type === 'I') の寸法グリッド：**

表示する寸法ラベル：
- H（全高）、B（フランジ幅）、t1（ウェブ厚）、t2（フランジ厚）、r1、r2

どちらも H形鋼と同じフィールド構成（H/B/t1/t2/r1/r2）なので、
`type === 'H' || type === 'C' || type === 'I'` の条件でまとめて処理できる。

---

### 5. SVG描画関数を追加する

#### 5-1. `drawChannelSVG(svgEl, spec, scale)` — 溝形鋼

H形鋼の SVG を参考に、片側フランジのみの断面を描く。

```
断面イメージ（左にウェブ、上下にフランジ）:

 ┌────────┐  ← 上フランジ（幅 B、厚 t2）
 │        │
 │        │  ← ウェブ（高さ H、厚 t1）
 │        │
 └────────┘  ← 下フランジ（幅 B、厚 t2）
```

- 左端にウェブ（幅 t1、高さ H）
- 上下に右向きフランジ（幅 B、厚 t2）
- 重心点 Cy を右向きに表示（x 軸方向のオフセット）

#### 5-2. `drawIBeamSVG(svgEl, spec, scale)` — I形鋼

H形鋼の SVG とほぼ同じ。両側対称のフランジとウェブ。
H形鋼のSVG描画関数 `drawHShapeSVG`（または同等の関数）をそのまま流用・コピーして `drawIBeamSVG` として定義し、`type === 'I'` のときに呼び出す。

### 6. `renderDataSpec()` の dispatch を更新する

```js
// 変更前（イメージ）
if (kindData.type === 'H') drawHShapeSVG(svgEl, spec, scale);
else if (kindData.type === 'L' || kindData.type === 'LU' || kindData.type === 'LUT') drawLAngleSVG(svgEl, spec, scale);
else if (kindData.type === 'FL') drawFlatBarSVG(svgEl, spec, scale);

// 変更後
if (kindData.type === 'H') drawHShapeSVG(svgEl, spec, scale);
else if (kindData.type === 'I') drawIBeamSVG(svgEl, spec, scale);
else if (kindData.type === 'C') drawChannelSVG(svgEl, spec, scale);
else if (kindData.type === 'L' || kindData.type === 'LU' || kindData.type === 'LUT') drawLAngleSVG(svgEl, spec, scale);
else if (kindData.type === 'FL') drawFlatBarSVG(svgEl, spec, scale);
```

また、塗装面積計算は：
- 溝形鋼 (type === 'C')：`(2*B + H - t2) / 1000` （外周 ≒ 上下フランジ外側 + ウェブ片側）
- I形鋼 (type === 'I')：`(4*B + 2*H - 2*t1 - 4*t2 + ...) / 1000`（H形鋼と同じ計算式を使用）

**I形鋼の塗装面積は H形鋼の `calcHPaintAreaPerMeter` をそのまま流用できる。**（フィールド構造が同じため）
