# コマンドパレット 修正指示

## 対象ファイル
- `index.html`
- `main.js`

---

## 修正内容

### 修正① 🔍が2つ表示される問題を解消

`index.html` 内の `id="cmdInput"` のplaceholderに🔍が含まれている場合は**削除する**。

**正しいplaceholder：**
```html
placeholder="例：H100、平9、丸32"
```

🔍はボタン側だけにある。inputのplaceholderには🔍を入れない。

---

### 修正② 入力時の候補を「H-100×50×5×7.6」形式で表示する

`main.js` の `cmdFilter()` 内、ドロップダウンアイテムのHTML生成部分を確認し、以下のように **規格名（spec）をそのまま全部表示する**ようにする。

候補の各行は：
- 左側：規格名フルで表示（例：`H-100×50×5×7.6`）
- 右側：kg/m値（例：`9.3 kg/m`）

```js
row.innerHTML = '<span>' + it.spec + '</span><span class="cmd-sub">' + it.kgm + ' kg/m</span>';
```

これはすでにこの形式になっているはずだが、もしkind名を省略していたり、specが切り詰められている場合は修正すること。

---

### 修正③ 重量ページも同じレイアウトにする

重量ページ（`id="wpp"`）のサイドバー内の鋼材選択エリアが、計算ページと全く同じ見た目（🔍ボタン左・入力欄右）になっていない場合は修正する。

`id="wCmdInput"` のplaceholderも🔍なしで：
```html
placeholder="例：H100、平9、丸32"
```

重量ページの `id="wCmdDropdown"` の候補表示も計算ページと同じく：
- 左：規格名フル（`H-100×50×5×7.6`）
- 右：kg/m値

---

## 確認事項

修正後、以下を確認すること：
1. 計算ページで🔍ボタンが1つだけ表示される（inputのplaceholderに🔍がない）
2. 入力欄に「h100」と打つと `H-100×50×5×7.6` 等の候補が出る
3. 重量ページも同じ見た目・動作になっている
4. 他の部分（履歴・在庫・詳細設定等）は一切変更しない
