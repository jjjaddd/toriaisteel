## 1. プロダクト概要

### プロダクト名
**TORIAI β**  
URL: https://jjjaddd.github.io/toriaisteel/

### 一言で言うと
鉄工所・作り物工場向けの**鋼材切断取り合い計算Webアプリ**。現在はβ版として公開中。

### 現在の実装機能
- 鋼材の取り合い計算（定尺から何本切れるかの最適化）
- 刃厚・端部ロスの設定
- 在庫定尺の管理（5,500〜12,000mm）
- 手持ち残材の登録・活用
- 切り出し部材リストの入力（Excel/CADからのタブ区切りコピペ対応）
- 切断履歴の記録 / 残材在庫管理
- 作業指示書の印刷・カート機能
- お問い合わせフォーム
- 重量計算シミュレーター ✅ 実装済み
- 塗装面積計算 ✅ 実装済み

## 3. 開発フロー（Claude → Codex → GitHub → 確認）

ClaudeはCowork（デスクトップアプリ）で動かす。デスクトップの `toriai` フォルダがGitHubと連携している（`とりあい` フォルダは別物・使わない）。

```
① オーナーがClaudeに「やりたいこと」を伝える
    ↓
② Claudeがデスクトップのtoriaiフォルダを自動で確認する
    ↓
③ ClaudeがCodexへの指示文を生成する
    ↓
④ オーナーがその指示文をCodexにそのまま貼る
    ↓
⑤ Codexがコードを生成し、toriaiフォルダを変更してGitHubにPUSH
    ↓
⑥ 次の作業時にClaudeが再びフォルダを確認して変更内容を把握する
```

### Claudeへの毎回の渡し方（テンプレート）

URLの貼り付けは不要。Coworkがフォルダを自動参照する。

```
【今日やりたいこと】
  ○○機能を追加したい

【詰まっていること（あれば）】
  F12のコンソールに出たエラー文をそのままコピペ
```

### 手動でやる必要があること
- Codexが生成したコードの動作確認（ブラウザで目視）
- GitHubへのPUSH確認（Codexが失敗した場合はオーナーが手動）

## 4. ターゲットユーザー

- **主対象**：鉄工所、作り物工場（町工場規模〜中規模）
- **業務内容**：鋼材を仕入れて切断・加工し製品を作る
- **現状のITリテラシー**：低め。10〜20年前のインストール型ソフトを使っていることが多い
- **競合ソフト例**：取合3・鉄人ダン・CUTPLAN・尺取虫Pro・よろずや2019

### TORIAIの勝ち筋
競合はインストール型・1PC1ライセンス・クラウドなし・チーム共有不可・スマホ非対応・UIが古い・高価（数万円〜）という弱点を持つ。

**TORIAIが目指す差別化：「どこでも・チームで・すぐ使える・安い」の4点を全部満たす唯一のツール**

## 5. プロダクトの全体構想（2層構造）

```
【層1】ツール（個人で使う）       ← まずここを固める
  └ 取合い計算            ✅ 実装済み
  └ 重量シミュレーター     ✅ 実装済み
  └ 塗装面積計算          ✅ 実装済み
  └ 鋼材規格検索（辞書）   → フェーズAで作る
  └ 見積もり即時回答       → フェーズAで作る
  └ カーボンフットプリント  → フェーズBで作る

【層2】データベース（みんなで育てる） ← ユーザーが集まってから
  └ 市中材流通体験談DB     → フェーズA（種まき）→ フェーズB（投稿機能）
  └ ミルシートデジタル管理  → フェーズB
  └ トレーサビリティ自動化  → フェーズB
  └ 端材マーケットプレイス  → フェーズB（登録のみ）→ フェーズC（売買）
  └ PDF図面自動読み込み    → フェーズC
```

**重要な原則：層1でユーザーを集めて、層2に誘導する順番を守る。層2は層1が成立しないと死んでいる。**

## 9. GitHub更新通知（Service Worker）

GitHubにPUSHされた時、ユーザーの画面に「新しい更新があります」と通知し、更新ボタンを押した瞬間にキャッシュを破棄してリロードする仕組み。**localStorageの在庫データには影響を与えない。**

### sw.js（新規作成）

```javascript
const CACHE_NAME = 'toriai-cache-v1';

self.addEventListener('install', (event) => {
  self.skipWaiting();
});

self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((cacheNames) => {
      return Promise.all(
        cacheNames
          .filter((name) => name !== CACHE_NAME)
          .map((name) => caches.delete(name))
      );
    }).then(() => self.clients.claim())
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    fetch(event.request).catch(() => caches.match(event.request))
  );
});

self.addEventListener('message', (event) => {
  if (event.data && event.data.type === 'SKIP_WAITING') {
    self.skipWaiting();
  }
});
```

### index.html に追加するコード（`</body>` 直前）

```html
<div id="sw-update-bar" style="
  display: none;
  position: fixed;
  bottom: 20px;
  left: 50%;
  transform: translateX(-50%);
  background: #2b3245;
  color: #fff;
  padding: 12px 24px;
  border-radius: 8px;
  box-shadow: 0 4px 16px rgba(0,0,0,0.3);
  z-index: 9999;
  align-items: center;
  gap: 16px;
  font-size: 14px;
  white-space: nowrap;
">
  <span>🔄 新しい更新があります</span>
  <button id="sw-update-btn" style="
    background: #3d63e8;
    color: #fff;
    border: none;
    padding: 6px 16px;
    border-radius: 6px;
    cursor: pointer;
    font-size: 14px;
  ">更新する</button>
</div>

<script>
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/toriaisteel/sw.js')
    .then((reg) => {
      reg.addEventListener('updatefound', () => {
        const newWorker = reg.installing;
        newWorker.addEventListener('statechange', () => {
          if (newWorker.state === 'installed' && navigator.serviceWorker.controller) {
            const bar = document.getElementById('sw-update-bar');
            bar.style.display = 'flex';
            document.getElementById('sw-update-btn').addEventListener('click', () => {
              caches.keys().then((names) => {
                return Promise.all(names.map((name) => caches.delete(name)));
              }).then(() => {
                newWorker.postMessage({ type: 'SKIP_WAITING' });
                window.location.reload();
              });
            });
          }
        });
      });
    })
    .catch((err) => console.error('SW登録失敗:', err));

  navigator.serviceWorker.addEventListener('controllerchange', () => {
    window.location.reload();
  });
}
</script>
```

**注意：登録パスはリポジトリ名に合わせること（例：`/toriaisteel/sw.js`）。localStorageはキャッシュAPIとは完全に別の領域なので在庫データは安全。**

## 11. 開発における注意事項

1. **「なんかうまく動く感じにして」はNG**　機能ごとに入力・出力・エラー処理を言語化してからCodexに渡す
2. **デバッグはClaudeとCodex両方に聞いて比較する**
3. **フェーズA完成後に一度コード整理の時間を取る**（AIが生成したコードは動くが散らかりやすい）
4. **認証・セキュリティはフェーズB以降・特に慎重に**（Supabaseの公式ドキュメントを必ず読む）
5. **一度に大量の変更をしない**（一つ作って動作確認してから次へ）

### 推奨技術スタック（フェーズB以降）
- バックエンド・DB：Supabase（無料枠あり・AIとの相性が良い）
