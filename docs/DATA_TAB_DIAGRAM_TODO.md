# データタブ断面図 TODO

データタブの断面図を、鋼種ごとの正式ひな形に差し替えていくための作業リスト。

目的は「数値を入れたら、見やすく整った断面図が自動でできる」状態にすること。
その場しのぎで SVG を拡大縮小するのではなく、鋼種ごとに図面の定義を決めてから実装する。

## 基本方針

- 鋼種ごとに、ユーザーが納品した SVG / HTML ひな形を正本にする
- TORIAI 側では、ひな形に `H` / `B` / `t1` / `t2` / `r1` / `r2` などの実データを流し込む
- 各鋼種の描画関数は `src/features/dataTab/sectionSvg.js` に集約する
- 鋼種データは既存の `src/data/steel/*/specs.js` を使う
- データ定義と描画ロジックを混ぜない
- レイアウト調整は `src/styles/dataPage.css` 側で行う
- 計算ロジック、Column Generation、Supabase Edge Functions には触らない

## 共通デザイン定義

全鋼種で揃える基準。

| 項目 | 方針 |
|---|---|
| 図の大きさ | TORIAI 側の白い図枠を基準にする。主要形状と寸法線が図枠の 78〜88% 程度を使う。SVG の viewBox 内比率だけで判断しない |
| 図の位置 | 枠の中央。ただし寸法線込みで視覚中央に置く |
| 線の太さ | 外形線、内形線、寸法線、補助線で太さを分ける |
| 寸法線の距離 | 図形に近すぎず、遠すぎない。文字と線が重ならない距離にする |
| 文字サイズ | デスクトップで読みやすい大きさを基準にする。最小 14px 相当を避ける |
| 文字位置 | 寸法線と干渉しない。図形の輪郭に被せない |
| 比率 | 実寸比を尊重。ただし極端に細長い鋼種は見やすさ優先で表示補正を許可 |
| 角丸 / R | ひな形に合わせる。簡略化しても形が崩れない範囲にする |
| 色 | 白 / 黒 / グレー基調。鋼材面は白、寸法線はグレー、重要文字は黒を基本にする |
| レスポンシブ | デスクトップ、タブレット、スマホで文字と寸法線が潰れない |

## 実装ルール

- ひな形を受け取ったら、まず対応鋼種の TODO を `Doing` に移す
- ひな形内の固定値を、実データから入る変数に置き換える
- 関数名は既存の `draw*SVG` に合わせる
- SVG 内に `NaN` / `undefined` が出ないようにチェックする
- 既存データにない値が必要な場合は、推測で補わず TODO に残す
- 一つの鋼種が終わったら、代表規格 3〜5 件で見た目を確認する
- 完了後に `service-worker.js` のキャッシュ名と `index.html` のバージョンを上げる

## 受け取りメモ

ひな形を受け取るときに見る項目。

- 対象鋼種
- 入力される寸法値
- ひな形内で固定されている寸法名
- 実寸比を守るか、見やすさ補正を入れるか
- R やテーパーなど、簡略化してよい部分
- 寸法線と文字の推奨位置
- 枠に対する図の推奨サイズ
